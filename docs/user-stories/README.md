# ユーザーストーリー管理ガイド

## 📝 ユーザーストーリーの記述形式

```
As a [ユーザータイプ]
I want to [実現したいこと]
So that [ビジネス価値・目的]
```

## ✅ 受け入れ基準（Acceptance Criteria）

各ストーリーには以下を含める：

1. **Given** - 前提条件
2. **When** - アクション
3. **Then** - 期待される結果

## 🏷️ ストーリーの分類

- **Epic**: 大きな機能グループ
- **Feature**: 具体的な機能
- **Story**: ユーザー視点のタスク
- **Task**: 技術的な実装タスク

## 📊 優先度レベル

- **P0**: Critical - ビジネス停止レベル
- **P1**: High - 主要機能
- **P2**: Medium - 改善・拡張
- **P3**: Low - Nice to have

## 🔄 ステータス

- **Draft**: 下書き
- **Ready**: 実装準備完了
- **In Progress**: 実装中
- **Testing**: テスト中
- **Done**: 完了
- **Blocked**: ブロック中
